多の関係になる方に紐付けるカラムが必要になる。

| カラム名 | 型 | 制約 | 説明 |
| :--- | :--- | :--- | :--- |
| id | INT | | マッチングID |
| user_id1 | INT | | ユーザーID|
|　user_id2 | INT |　|　ユーザーID |

## API エンドポイント (JSON)

APIとしては機能はさせるが、画面は作成しない次に持ってくいくか
### 1.ユーザー登録　
- メソッド：POST
- URL:/users
- 認証：不要
- <mark>リクエストBody</mark>{
  "email":"メールアドレス",
  "password":"パスワード"
  }
- レスポンス例：{
  "id":1,
  "email":"test@gmail",
  "created_at":"2026/05/17"
}

### 2.ログイン
- メソッド：POST
- /login
- 認証：必要
- <mark>リクエストBody</mark>{
  "email":"メールアドレス",
  "password":"パスワード"
}

### プロフィール画面を取得
- メソッド：GET
- URL:/users/{user_id}
- 認証：必要
- レスポンス例：{
  "user_id":1,
  "penname":"バクマン",
  "introduction":"私はすごく仕事をするよ"
  "personality":"いい加減な性格"
  "work":"https://ssss.com"
}

### マッチング検索画面
- GET
URL:/users/user
- 認証：必要
- レスポンス例：[{
  "user_id":1,
  "penname":"バクマン",
  "introduction":"私はすごく仕事をするよ"
  "personality":"いい加減な性格"
  "work":"https://ssss.com"
}]

### マッチング登録
- メソッド：POST
- URL:/matches
- 認証：必要
- リクエストBody{
  "user_id1":1,
  "user_id2":2
}
- レスポンス例：{
  201 Created
}

### マッチング画面を取得
- メソッド：GET
- URL:/matches
- 認証：必要
- レスポンス例{
  "matching_id":1,
  "user_id1":1
  "user_id2":2
}
